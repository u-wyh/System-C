# 系统 C++ 阶段三 — 系统优化与云原生（21 天）

> 阶段目标：
> 从“能并发、稳定运行的服务” → “工业级系统 + 性能优化 + 云部署能力”
>
> 核心能力：
> - 内存和 CPU 性能分析
> - 高并发优化
> - Linux 系统与内核理解
> - 云原生服务设计（容器化 / 弹性 / 日志监控）
> - 高可用、可扩展系统设计
> - 工程化与自动化能力

---

## 第 1 周：性能分析与优化

### Day 1 — 内存与 CPU 性能测量
**目标**：学会量化性能
**任务**：
- 用 `perf`, `valgrind`, `gdb` 等工具分析你的服务
- 测试 CPU 使用率、内存占用、内存泄漏
**达标效果**：
- 能画出服务 CPU / 内存瓶颈图
- 能解释哪个模块占用资源最多

---

### Day 2 — 内存管理优化
**目标**：减少内存碎片和泄漏
**任务**：
- 分析 Resource 类（File / Socket / Thread）内存分配
- 使用智能指针替代裸指针
- 检查异常路径是否存在泄漏
**达标效果**：
- 内存泄漏 0
- 异常路径资源安全释放

---

### Day 3 — CPU 优化
**目标**：提高服务吞吐量
**任务**：
- 使用 CPU Profiling 找出热点函数
- 对热点函数进行优化：
  - 减少锁竞争
  - 减少系统调用
  - 避免频繁分配释放内存
**达标效果**：
- 吞吐量明显提升
- CPU 占用降低

---

### Day 4 — 高并发优化
**目标**：线程池 + IO 调度提升性能
**任务**：
- 对线程池参数（线程数、队列大小）做实验
- 观察 epoll + ThreadPool 的吞吐量变化
- 尝试 batch 提交任务优化
**达标效果**：
- 并发量提升
- 任务延迟下降
- 系统稳定

---

### Day 5 — 半包 / 粘包优化
**目标**：网络数据处理稳定
**任务**：
- 完善 TCP Server 粘包处理逻辑
- 写测试客户端模拟高频小包
**达标效果**：
- 不丢包
- 任务延迟可控

---

### Day 6 — 异常与资源回收优化
**目标**：长期运行无崩溃
**任务**：
- 模拟线程异常、IO 异常、内存分配失败
- 观察服务资源释放情况
**达标效果**：
- 服务长时间运行稳定
- 日志记录完整

---

### Day 7 — 周总结
**目标**：性能和稳定性总结
**任务**：
- 写总结：
  - 性能瓶颈定位方法
  - 高并发优化经验
- 画性能对比图
**达标效果**：
- 能口头讲清楚优化思路

---

## 第 2 周：Linux 系统与内核理解

### Day 8 — Linux 文件系统和 IO
**目标**：理解文件、Socket 内核处理
**任务**：
- 分析 open / read / write / close 系统调用
- 使用 `strace` 跟踪服务行为
**达标效果**：
- 能解释每个系统调用耗时原因
- 知道 IO 阻塞的底层机制

---

### Day 9 — 进程和线程模型
**目标**：系统级线程理解
**任务**：
- 理解 PID、TID、线程调度
- 查看 `/proc` 下线程状态
**达标效果**：
- 能解释线程调度、CPU 核心绑定
- 能理解多核下的性能瓶颈

---

### Day 10 — 内核网络栈
**目标**：理解 TCP/IP 内核处理
**任务**：
- 使用 `netstat`, `ss` 分析连接状态
- 学习 TCP 三次握手、四次挥手
- 理解 socket buffer 作用
**达标效果**：
- 能口头讲清 TCP 连接全过程
- 能解释拥塞与流控

---

### Day 11 — epoll 原理与优化
**目标**：理解 epoll 内核实现
**任务**：
- 学习 epoll LT / ET 的区别
- 分析红黑树 + 就绪队列结构
- 尝试优化事件处理顺序
**达标效果**：
- 能画 epoll 内核数据结构
- 能解释为什么 ET 可以减少系统调用

---

### Day 12 — 线程竞争与 NUMA
**目标**：高并发深入优化
**任务**：
- 模拟线程锁竞争
- 使用 `numactl` 测试 NUMA CPU 分配
**达标效果**：
- 线程调度优化
- 数据局部性优化

---

### Day 13 — 日志与监控系统
**目标**：工程化能力
**任务**：
- 完善日志系统（异步 + 分级 + 文件滚动）
- 增加简单监控接口（CPU / 内存 / 连接数）
**达标效果**：
- 可在线观察系统状态
- 日志能长期记录运行状况

---

### Day 14 — 周总结
**目标**：内核和系统理解总结
**任务**：
- 写阶段总结：
  - IO / TCP / epoll / 线程调度知识
- 画系统数据流图
**达标效果**：
- 能在面试中讲清楚系统运行原理
